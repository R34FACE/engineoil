# エンジンオイル 粘度別販売数集計

「単品別売上集計リスト」のPDFをブラウザ内でOCRし、エンジンオイルを **粘度 → 商品 → 販売数** の順で集計するGitHub Pages用サイトです。

容量の合計は計算しません。缶商品は「缶」、量り売り商品は「件」で表示します。

## 主な機能

- 複数PDF・画像をまとめて読み込み
- 商品コード・商品名・売上数量を自動抽出
- `0W20`、`OW-20`、`0W-20`などを`0W-20`へ統一
- 粘度別 → 商品別に販売数を集計
- 商品名・粘度・販売数を手動修正
- 修正した商品名を商品コードごとにブラウザへ記憶
- 商品名・粘度・商品コード検索
- 量り売り商品の表示切替
- 粘度×商品CSV、読み取り明細CSV、バックアップJSON
- PDFはブラウザ内で処理

## GitHub Pagesへの公開

1. GitHubで新しいリポジトリを作成します。
2. このフォルダ内の `index.html`、`styles.css`、`app.js`、`.nojekyll` をリポジトリ直下へアップロードします。
3. `Settings` → `Pages` を開きます。
4. `Source` を `Deploy from a branch` にします。
5. Branchを `main`、Folderを `/(root)` にして保存します。

既存のタイヤ集計サイトと同じ構成です。上書きしたくない場合は、新しいリポジトリを作成してください。

## ローカル確認

```bash
python -m http.server 8000
```

ブラウザで `http://localhost:8000` を開きます。

## 初期の読み取り設定

- 売上数量列の中心位置：87.5%
- 数量列の許容幅：3.0%
- 商品名欄の右端：78.5%
- 同じ行とみなす高さ：0.9%

別レイアウトで単価や金額を数量として拾う場合は、数量列の中心位置と許容幅を調整してください。

## 使用ライブラリ

- PDF.js 6.0.227
- Tesseract.js 7.0.0

初回はライブラリと日本語OCRデータを取得するため、インターネット接続が必要です。

## 注意

OCRでは `0` と `O`、`1` と `I`、商品名の一部を誤認識する場合があります。集計後に「読み取り明細・修正」タブで確認してください。
